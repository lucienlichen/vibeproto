1. 项目名称

VibeCoding 原型管理系统

2. 项目背景

当前团队在 B/S 架构项目中，常通过 AI 编程工具快速生成 Web 前端原型或 HTML 演示页面，用于需求评审、领导汇报和客户演示。
但现有方式存在以下问题：

原型项目散落在个人电脑、本地目录或临时服务器中，难以统一管理；

原型版本缺少可视化记录，后续修改容易混乱；

原型部署依赖手工操作，效率低；

历史版本难以回溯和回滚；

需求文档、提示词、源码、演示地址之间没有建立关联；

缺乏标准化的 vibecoding 原型交付流程。

因此需要建设一个原型管理系统，统一管理 AI 生成的原型项目，实现快速纳管、快速部署、版本可追踪、原型可回滚、资产可沉淀。

3. 项目目标

建设一个服务于 vibecoding 工作流的原型管理平台，实现以下目标：

支持快速创建原型项目；

支持上传 HTML/前端工程源码；

支持自动构建与部署；

支持发布版本管理；

支持 latest 固定预览地址；

支持历史版本访问和一键回滚；

支持保存需求文档与 AI 提示词；

支持团队内统一原型交付流程。

5. 业务范围
5.1 支持的原型类型

纯静态 HTML 原型；

Vue 前端工程原型；

React 前端工程原型；

后续可扩展其他静态部署型项目。

5.2 支持的代码导入方式

上传 zip 包；

Git 仓库拉取；

在线快速创建 HTML 页面。

5.3 支持的部署方式

构建为静态站点并发布；

为每个版本生成独立访问地址；

latest 地址始终指向当前展示版本；

支持历史版本访问和回滚。

6. 业务流程
6.1 项目创建

用户进入项目管理；

创建项目；

选择项目类型；

填写名称、标识、简介；

选择默认构建模板；

系统初始化项目空间。

6.2 源码导入

选择代码导入方式；

上传 zip / 配置 Git / 输入 HTML；

生成源码版本记录；

填写版本说明。

6.3 构建发布

选择源码版本；

点击构建发布；

系统创建构建任务；

执行安装依赖、构建、校验；

构建成功后生成发布版本；

生成访问地址；

可选择设为 latest。

6.4 历史回滚

查看历史发布版本；

选择目标版本；

点击“设为当前版本”；

latest 指向目标版本；

记录回滚日志。

6.5 AI 协同迭代

在项目中维护需求和提示词；

将文档投喂给 AI 工具；

AI 输出代码；

用户导入代码到本系统；

系统自动构建发布；

版本与需求、提示词建立关联。
